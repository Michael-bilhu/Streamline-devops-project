What This Project Demonstrates

Linux Foundation

Project managed from Ubuntu terminal using Linux commands, file navigation, SSH concepts, and command-line workflows.

Docker Containerization

Frontend and backend applications are packaged into Docker images for consistent deployment across environments.

Docker Hub Registry

Application images are pushed to Docker Hub so Kubernetes can pull versioned deployment artifacts.

Kubernetes Deployments

Deployments manage desired state, replicas, pod lifecycle, rolling updates, and self-healing behavior.

Kubernetes Services

Services provide stable networking so frontend pods can communicate with backend pods without using unstable pod IPs.

Ingress Routing

Ingress routes external traffic into Kubernetes services, similar to a production reverse proxy.

Jenkins CI/CD

Jenkins automates the build, push, and Kubernetes rollout process instead of requiring manual deployment commands.

Self-Healing Infrastructure

Kubernetes automatically recreates pods when they fail, keeping the desired number of replicas running.
